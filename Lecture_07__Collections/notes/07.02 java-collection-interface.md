## Collection Interface in Java

This module explores the `Collection` interface in Java, a fundamental part of the Java Collections Framework (JCF). It represents a group of objects, referred to as elements.  Understanding the `Collection` interface is crucial for effectively using the various data structures provided by the JCF.

### 1. Introduction to the `Collection` Interface

The `Collection` interface resides in the `java.util` package and serves as the root interface for most collection classes in Java.  It provides core functionalities for working with groups of objects, without specifying the underlying implementation details (e.g., whether the collection is ordered, allows duplicates, etc.).  This abstraction allows for flexibility and code reusability.

Key features of the `Collection` interface:

* **Abstraction:**  Defines common methods applicable to various collection types.
* **Polymorphism:** Allows treating different collection types uniformly through the `Collection` interface.
* **Flexibility:** Enables switching between different collection implementations without significant code changes.
* **Extensibility:** Provides a foundation for creating custom collection implementations.

### 2. Core Methods of the `Collection` Interface

The `Collection` interface declares several essential methods:

* `int size()`: Returns the number of elements in the collection.
* `boolean isEmpty()`: Checks if the collection is empty.
* `boolean contains(Object o)`: Checks if the collection contains a specific element.
* `boolean add(E e)`: Adds an element to the collection.
* `boolean remove(Object o)`: Removes a specific element from the collection.
* `Iterator<E> iterator()`: Returns an iterator over the elements in the collection.
* `Object[] toArray()`: Converts the collection to an array.
* `<T> T[] toArray(T[] a)`: Converts the collection to an array of a specified type.
* `boolean containsAll(Collection<?> c)`: Checks if the collection contains all elements of another collection.
* `boolean addAll(Collection<? extends E> c)`: Adds all elements of another collection to this collection.
* `boolean removeAll(Collection<?> c)`: Removes all elements of another collection from this collection.
* `boolean retainAll(Collection<?> c)`: Retains only the elements in this collection that are also present in another collection.
* `void clear()`: Removes all elements from the collection.


### 3. Code Examples

```java
import java.util.ArrayList;
import java.util.Collection;
import java.util.HashSet;

public class CollectionExample {
    public static void main(String[] args) {
        // Using ArrayList (ordered collection)
        Collection<String> list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Orange");
        System.out.println("List: " + list); // Output: List: [Apple, Banana, Orange]

        // Using HashSet (unordered, no duplicates)
        Collection<String> set = new HashSet<>();
        set.add("Apple");
        set.add("Banana");
        set.add("Apple"); // Duplicate, won't be added
        System.out.println("Set: " + set); // Output: Set: [Apple, Banana]

        // Iterating through a collection
        for (String fruit : list) {
            System.out.println(fruit);
        }


        // Checking if an element exists
        if (list.contains("Banana")) {
            System.out.println("Banana is present in the list");
        }


        // Removing an element
        list.remove("Banana");
        System.out.println("List after removal: " + list); // Output: List after removal: [Apple, Orange]
    }
}
```

### 4. Key Considerations

* **Choosing the right implementation:** The choice of `Collection` implementation (e.g., `ArrayList`, `HashSet`, `LinkedList`) depends on the specific requirements of your application, such as whether order is important, whether duplicates are allowed, and the frequency of various operations (e.g., insertion, deletion, search).

* **Generics:**  Use generics to specify the type of elements stored in the collection to improve type safety and avoid runtime errors.
