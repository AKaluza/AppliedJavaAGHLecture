## Java Programming: Delving into Classes and Objects

Welcome to the world of Object-Oriented Programming (OOP)! In this session, we'll explore the fundamental building blocks of OOP in Java: **classes and objects**. 

### Understanding the Paradigm Shift

Before we dive into the syntax, let's grasp the core concept. Unlike procedural programming, which focuses on procedures and data as separate entities, OOP emphasizes bundling data and the methods that operate on that data into self-contained units called **objects**.

Think of it like this: 

* **Procedural Programming:** Building a house brick by brick, step by step.
* **Object-Oriented Programming:**  Designing a blueprint (class) for a house and then creating multiple instances (objects) from that blueprint.

### What is a Class?

In essence, a **class** is a blueprint or a template that defines the structure and behavior of a particular type of object. It encapsulates:

* **Data (Attributes/Fields):** Variables that hold the object's state or properties.
* **Methods (Behavior):** Functions that define the actions an object can perform.

**Example:** Let's say we want to represent a "Dog" in our code. We can create a "Dog" class:

```java
class Dog {
    // Attributes
    String breed;
    String name;
    int age;

    // Methods
    void bark() {
        System.out.println("Woof!");
    }

    void eat(String food) {
        System.out.println(name + " is eating " + food);
    }
}
```

In this example:

* `breed`, `name`, and `age` are attributes describing a dog's characteristics.
* `bark()` and `eat()` are methods defining a dog's actions.

### What is an Object?

An **object** is an instance of a class. It's the actual realization of the blueprint defined by the class. You can create multiple objects from the same class, each having its own distinct state.

**Example:**

```java
public class Main {
    public static void main(String[] args) {
        // Create two Dog objects
        Dog myDog = new Dog();
        Dog yourDog = new Dog();

        // Set attributes
        myDog.breed = "Labrador";
        myDog.name = "Buddy";
        myDog.age = 3;

        yourDog.breed = "Golden Retriever";
        yourDog.name = "Max";
        yourDog.age = 5;

        // Invoke methods
        myDog.bark(); // Output: Woof!
        yourDog.eat("bones"); // Output: Max is eating bones
    }
}
```

Here, `myDog` and `yourDog` are two different `Dog` objects with their own unique values for breed, name, and age.

### Key OOP Concepts Illustrated:

* **Abstraction:**  The `Dog` class represents the essential qualities of a dog without getting bogged down in unnecessary details.
* **Encapsulation:** Data and methods are bundled together within the `Dog` class, promoting data protection.
* **Inheritance:** (Covered in a later session) - Allows you to create new classes (subclasses) based on existing ones, inheriting their properties and behaviors.
* **Polymorphism:** (Covered in a later session) -  Allows objects of different classes to be treated as objects of a common type.
